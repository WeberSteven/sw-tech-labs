Auto Deploy via Signed Webhook

This environment uses a segmented multi-tier architecture. Cloudflare provides DNS and SSL termination at the edge. An internal Docker-based nginx reverse proxy routes traffic securely to a dedicated web server tier.

Code changes are deployed through a GitHub webhook pipeline with HMAC signature validation, triggering an automated git-based deployment process. The webhook endpoint is firewall-restricted and only accessible from the reverse proxy.

This setup mirrors modern production design patterns including layered security, internal segmentation, and automated CI/CD workflows.
